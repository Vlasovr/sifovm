Практический пакет для модуля стека, вариант 4.

Состав:
1) stack7x16_standalone.vhd      - отдельный модуль стека на 7 слов по 16 бит
2) tb_stack7x16_standalone.vhd   - testbench с assert-проверками

Логика работы:
- глубина стека: 7 слов по 16 бит;
- рост стека: вниз;
- после сброса SP = 7, стек пуст;
- PUSH записывает слово и уменьшает SP;
- POP выдает верхнее слово в dout_o и увеличивает SP;
- при переполнении PUSH игнорируется;
- при опустошении POP игнорируется;
- одновременные push_i='1' и pop_i='1' трактуются как no-op.

Что смотреть на временной диаграмме:
- clk
- rst
- push
- pop
- din
- dout
- sp
- empty
- full

Ожидаемое поведение:
- после reset: sp=7, empty=1, full=0;
- после 7 PUSH: sp=0, full=1;
- при POP данные выходят в обратном порядке (LIFO);
- после 7 POP: sp=7, empty=1.

Как запускать:
1. Создай проект в Quartus или открой симуляцию в ModelSim.
2. Добавь оба файла .vhd.
3. Назначь top-level для симуляции: tb_stack7x16_standalone.
4. Запусти симуляцию примерно на 200 ns.
5. Если все хорошо, в transcript появится сообщение:
   tb_stack7x16_standalone: TEST PASSED

Если преподаватель требует другую реакцию на одновременный PUSH+POP,
это легко меняется в основном модуле в единственном месте процесса.
