# Финальный чеклист перед сдачей

## Что уже подготовлено в комплекте

- VHDL-модули всех основных блоков.
- Полный top-level `system_coursework_top`.
- Сквозной testbench `tb_system_coursework`.
- MIF-образы ROM/RAM.
- Ожидаемый дамп RAM после `HLT`.
- Редактируемые схемы A1-A4 и 16 отдельных PDF-листов.
- Таблица сигналов.
- Листинг программы в символьном, hex и binary виде.
- Инструкция для Quartus/ModelSim.

## Что нужно сделать вручную в Quartus/ModelSim

1. Открыть/создать проект.
2. Скомпилировать файлы в порядке из `QUARTUS_MODELSIM_RUNBOOK.md`.
3. Запустить `tb_system_coursework`.
4. Проверить строку `TEST PASSED`.
5. Снять временные диаграммы.
6. Снять дампы RAM/ROM/cache.
7. Вставить готовые PDF-схемы из `schemes/pdf/`: минимум 4 основных листа, при необходимости добавить детальные.
8. Вставить реальные скриншоты в записку.

## Что нужно проверить в записке

- Есть лист задания с вариантом 4.
- Введение описывает именно индивидуальное задание.
- Все операции АЛУ: `INCS`, `OR`, `NOR`, `SRA`.
- Везде 12 РОН, а не 8.
- Везде 16-битные шины.
- Стек 7 слов, рост вниз.
- Предсказатель A4, а не A2/A3/A5.
- DMA: адрес 10, объем 6 байт.
- Кэш: k=4, замещение по наибольшей давности хранения.
- Есть листинг программы в hex и binary.
- Есть общий результат моделирования всей системы.
- Есть дампы ROM/RAM/cache.

## Что лучше говорить на защите

- Проект построен по гарвардской архитектуре, поэтому тракт команд и тракт данных разделены.
- Все команды имеют двухсловный формат, что упрощает дешифрацию и фазовый автомат.
- Кэш находится между CPU data port и RAM, запись сквозная.
- При одновременном запросе CPU и DMA арбитр выдает приоритет DMA.
- `JZ` добавлена для демонстрации предсказателя A4.
- Тестовая программа специально проходит через АЛУ, стек, память, переходы, DMA и `HLT`.
