# Курсовой проект СиФО ЭВМ, вариант 4

## Что нужно сдать

По методичке проект состоит из трех основных частей:

1. Пояснительная записка не менее 30 страниц, Times New Roman 14, одинарный интервал.
2. Графическая часть: минимум 4 листа формата А3.
3. Исходники проекта для Quartus/ModelSim, временные диаграммы и дампы памяти.

Критически важно: в записке должен быть листинг программы, который включает все команды архитектуры, в символьном и бинарном виде. Без этого курсовой могут не принять.

## Параметры варианта 4

- Архитектура: гарвардская.
- Шины: адрес 16 бит, данные 16 бит.
- Память: синхронное ПЗУ команд и синхронное ОЗУ данных.
- Адресация: прямо-регистровая плюс обязательная прямая адресация памяти.
- РОН: 12 регистров по 16 бит.
- АЛУ: INCS, OR, NOR, SRA.
- Обязательные команды: MOV adr, reg; MOV reg, adr; JMP adr; PUSH reg; POP reg; HLT.
- Для демонстрации предсказателя в проект добавлена команда JZ adr.
- Стек: 7 слов по 16 бит, рост вниз.
- Кэш: k = 4, замещение по наибольшей давности хранения, сквозная запись с отображением.
- Предсказатель переходов: A4, индекс PC(2) || GHR(2), PHT + BTB + GHR.
- КПДП/DMA: начальный адрес 10, объем 6 байт, то есть 3 слова по 16 бит.
- Арбитраж шин: централизованный параллельный, приоритет DMA при одновременных запросах.

## Листы, которые лежат в этой папке

- `A1_general_structure.drawio` - общая структурная схема микро-ЭВМ.
- `A2_control_unit.drawio` - устройство управления.
- `A3_datapath_alu_stack.drawio` - исполнительный тракт: РОН, АЛУ, FR, стек.
- `A4_memory_dma_arbiter.drawio` - память, кэш, DMA/КПДП, арбитр, предсказатель.

Файлы сделаны как редактируемые черновики для diagrams.net/draw.io. Их нужно открыть, выровнять связи, добавить обозначения шин, номера рисунков и оформить рамку/штамп под требования преподавателя.

## Ближайший порядок работы

1. Довести лист A1 до аккуратного вида: крупные блоки, 16-битные шины, направления обмена, управляющие связи пунктиром.
2. После A1 привести A3, потому что АЛУ, РОН и стек уже имеют VHDL-заготовки и тестбенчи.
3. Потом сделать A4: кэш можно пока показывать структурно, но DMA и арбитр должны быть связаны конкретными REQ/GNT/RAM_WE/RAM_ADDR/RAM_DATA.
4. Последним делать A2: по нему удобнее сверяться с уже готовыми листами A1/A3/A4.
5. В ModelSim/Quartus снять минимум временные диаграммы: ALU, стек, ROM/RAM, DMA+арбитр, предсказатель A4.

## Что проверить в записке

`Курсовая/Записка Власов.docx` уже похожа на рабочую основу, но перед сдачей ее надо вычистить:

- заменить упоминание Quartus 9.1 на версию из требований, если преподаватель строго держится диапазона 7.11-9.0;
- убрать старый фрагмент про 8-битное АЛУ и операции ADDC/AND/NOT/ROL/CMP;
- заменить чужие/референсные временные диаграммы на собственные;
- добавить в приложение финальные схемы и реальные дампы ROM/RAM/cache;
- убедиться, что все рисунки и таблицы пронумерованы единым стилем.
