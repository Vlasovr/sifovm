# Дампы памяти

## ROM

Файл: `memory/rom_variant4.mif`.

ROM содержит тестовую программу из `docs/PROGRAM_LISTING.md`. Команды занимают адреса `0000h..0021h`, остальные слова равны `0000h`.

## RAM до запуска

Файл: `memory/ram_variant4_initial.mif`.

| Адрес | Значение | Назначение |
|---:|---:|---|
| `000A` | `0000` | цель DMA, слово 0 |
| `000B` | `0000` | цель DMA, слово 1 |
| `000C` | `0000` | цель DMA, слово 2 |
| `0020` | `8001` | исходное значение для `SRA` |
| `0021` | `00F0` | первый операнд логических операций |
| `0022` | `0F0F` | второй операнд для `OR` |
| `0023` | `0000` | операнд для `NOR` |
| `0024` | `1234` | контрольное слово для пропущенных команд |
| `0025` | `FFFF` | операнд для получения нуля |
| `0030` | `0000` | ячейка результата |

## RAM после HLT

Файл: `memory/ram_variant4_expected_after_hlt.mif`.

| Адрес | Значение | Почему изменилось |
|---:|---:|---|
| `000A` | `1111` | первое слово DMA |
| `000B` | `2222` | второе слово DMA |
| `000C` | `3333` | третье слово DMA |
| `0030` | `C001` | результат `SRA + INCS`, прошедший через стек |

Остальные контрольные слова должны сохраниться.

## Что показать в записке

На скриншоте RAM/dump после моделирования должны быть видны минимум адреса:

```text
000A, 000B, 000C, 0020, 0021, 0022, 0023, 0024, 0025, 0030
```

Отдельно стоит подписать, что адреса `000A..000C` изменены DMA, а адрес `0030` изменен командой `MOV R3, 0030h`.
