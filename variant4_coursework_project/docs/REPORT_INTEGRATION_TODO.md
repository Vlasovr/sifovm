# Что вставить в пояснительную записку

Базовый файл записки в текущем репозитории:

```text
Курсовая/Записка Власов.docx
```

Он уже похож на рабочую основу, но перед сдачей нужно синхронизировать его с этим комплектом.

## Обязательные замены

1. Убрать старый фрагмент про 8-битное АЛУ и операции `ADDC`, `AND`, `NOT`, `ROL`, `CMP`.
2. Везде держать параметры варианта 4:
   - 16-битный адрес;
   - 16-битные данные;
   - 12 РОН;
   - `INCS`, `OR`, `NOR`, `SRA`;
   - стек 7 слов вниз;
   - A4 `PC(2)||GHR(2)`;
   - DMA с адреса 10 и объемом 6 байт.
3. Заменить референсные изображения временных диаграмм на собственные скриншоты из ModelSim/Quartus.
4. Вставить четыре финальных листа из `schemes/`.
5. Вставить листинг из `docs/PROGRAM_LISTING.md`.
6. Вставить MIF/дампы из `memory/`.

## Минимальный набор рисунков в ПЗ

- общая структурная схема;
- схема устройства управления;
- схема исполнительного тракта;
- схема памяти/DMA/арбитра;
- временная диаграмма АЛУ;
- временная диаграмма стека;
- временная диаграмма DMA/арбитра;
- временная диаграмма полной системы;
- дамп RAM до и после выполнения;
- при возможности участок обновления предсказателя A4.

## Контроль перед печатью

- Нумерация рисунков идет единым стилем.
- Нумерация таблиц идет единым стилем.
- В списке литературы есть методичка и материалы `1.BO6`, `2.ALU6`, `3.CUU6`, `4.MPUU6`, `5.ADDRESS`.
- В заключении перечислены именно параметры варианта 4.
- Нет упоминаний чужого варианта 11.
- Нет утверждения о фактической Quartus-компиляции, пока ты ее реально не сделал.
