# Графическая часть

В каталоге `schemes/` лежат четыре редактируемых листа `.drawio`.

| Лист | Файл | Что показывает |
|---|---|---|
| А1 | `A1_general_structure.drawio` | общая структурная схема микро-ЭВМ |
| А2 | `A2_control_unit.drawio` | устройство управления, дешифратор, секвенсор фаз и матрица сигналов |
| А3 | `A3_datapath_alu_stack.drawio` | РОН, АЛУ, регистр флагов, стек, тракт записи результата |
| А4 | `A4_memory_dma_arbiter.drawio` | ROM/RAM, кэш, DMA/КПДП, арбитр, предсказатель переходов |

## Что нужно сделать перед сдачей

1. Открыть каждый файл в diagrams.net или draw.io.
2. Проверить, что все стрелки читаются и не перекрывают подписи.
3. Добавить рамку и основной штамп, если преподаватель требует ЕСКД-оформление.
4. Экспортировать каждый лист в PNG/PDF.
5. Вставить экспортированные изображения в пояснительную записку и приложения.

## Что обязательно должно быть видно

### А1

- ПЗУ команд;
- IP/IR;
- устройство управления;
- РОН 12 x 16;
- АЛУ `OR/NOR/SRA/INCS`;
- регистр флагов;
- стек 7 x 16 с ростом вниз;
- кэш данных;
- ОЗУ данных;
- предсказатель A4;
- DMA/КПДП;
- арбитр шины.

### А2

- дешифратор команды;
- секвенсор `T0...T5 + Tw`;
- входы `FR.Z`, `FR.S`, `FR.C`, `FR.O`;
- события `CACHE_MISS`, `DMA_WAIT`, `HLT`;
- матрица управляющих сигналов;
- логика загрузки IP.

### А3

- два канала чтения РОН;
- один канал записи РОН;
- MUX входов АЛУ;
- ALU result writeback;
- формирование `Z/S/C/O`;
- `PUSH`/`POP` и изменение `SP`.

### А4

- отдельный ROM-тракт команд;
- RAM-тракт данных;
- `REQ_CPU/GNT_CPU`;
- `REQ_DMA/GNT_DMA`;
- кэш `4-way, 16 sets`;
- замещение по максимальному возрасту строки;
- сквозная запись;
- запись DMA по адресам `000A..000C`;
- предсказатель A4 с `PHT + BTB + GHR`.
