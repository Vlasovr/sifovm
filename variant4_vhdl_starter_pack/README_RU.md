# VHDL starter pack для курсового, вариант 4

В этом наборе есть две группы файлов:

- `src/` - исходники блоков и верхних заготовок;
- `tb/` - простые тестбенчи для ModelSim/Questa.

## Что здесь уже можно использовать как основу

Полезные и почти готовые модули:

- `alu_variant4.vhd` - операции OR, NOR, SRA, INCS;
- `reg_file12x16.vhd` - 12 регистров по 16 бит;
- `flags_reg.vhd` - регистр флагов Z/S/C/O;
- `stack7x16.vhd` - стек глубиной 7 слов с ростом вниз;
- `rom_sync.vhd` - синхронное ПЗУ с тестовой программой из записки;
- `ram_sync.vhd` - синхронное ОЗУ с начальными данными;
- `branch_predictor_a4.vhd` - предсказатель A4 на PHT/BTB/GHR;
- `bus_arbiter_2master.vhd` - арбитр CPU/DMA с приоритетом DMA;
- `dma_controller_3word.vhd` - контроллер КПДП на 3 слова.

Заготовки, которые надо будет дорабатывать под итоговую модель:

- `cache_controller_stub.vhd` - только интерфейс и комментарии по логике;
- `top_variant4_stub.vhd` - каркас верхнего уровня.

## Важное ограничение

Этот пакет не проверялся компиляцией в Quartus/ModelSim в текущей среде, потому что соответствующих VHDL-инструментов здесь нет. Поэтому относись к нему как к аккуратной стартовой базе, а не как к гарантированно собранному проекту.

## Как запускать в ModelSim

1. Создай проект и добавь все файлы из `src/`.
2. После этого добавь нужный testbench из `tb/`.
3. Для быстрой проверки по отдельным узлам запускай:
   - `tb_alu_variant4`
   - `tb_stack7x16`
   - `tb_mem_dma`
4. На диаграммы выведи сигналы, которые уже перечислены в пояснительной записке:
   - для АЛУ: `a`, `b`, `op`, `y`, `z`, `s`, `c`, `o`;
   - для стека: `push`, `pop`, `din`, `dout`, `sp`;
   - для памяти и DMA: `start`, `dev_data`, `ram_we`, `ram_addr`, `ram_din`, `ram_dout`, `dma_done`.

## Как это связано с графической частью

- `alu_variant4.vhd` соответствует листу А3 по АЛУ/РОН;
- `stack7x16.vhd` - листу стека;
- `rom_sync.vhd` и `ram_sync.vhd` - листам памяти;
- `branch_predictor_a4.vhd`, `dma_controller_3word.vhd`, `bus_arbiter_2master.vhd` - листу периферийной подсистемы и управления памятью;
- `top_variant4_stub.vhd` можно использовать как основу верхней структурной схемы.

## Что лучше делать дальше

1. Сначала собрать и проверить АЛУ, РОН, стек, ROM и RAM.
2. Потом подключить DMA и арбитр.
3. Потом добавить предсказатель переходов.
4. Кэш оставлять на конец, потому что это самый капризный блок.
